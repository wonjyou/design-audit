# UX Copy Reference

## Core Principles
- **Plain language first**: Write at a Grade 8 reading level unless domain demands otherwise
- **Active voice**: "Save your changes" not "Changes will be saved"
- **Second person**: Speak directly to the user ("you", "your")
- **Outcome-focused CTAs**: The button label should describe what happens after clicking, not the action itself
  - ✅ "Start Free Trial" → ❌ "Submit"
  - ✅ "Delete Account" → ❌ "OK"
  - ✅ "Save and Continue" → ❌ "Next"

## Call-to-Actions (CTAs)
- Primary CTA: verb + noun (specific outcome): "Create Project", "Send Message", "Book Appointment"
- Avoid: "Click Here", "Learn More" (without context), "Submit", "OK", "Yes/No"
- Button copy should set correct expectations (don't say "Free" if a CC is required)
- Confirm before destructive actions: "Delete Project" button → "Yes, delete this project" in confirmation

## Labels
- Form field labels: noun-first, title case or sentence case (be consistent)
- Navigation labels: should be mutually exclusive and exhaustive
- Section headers: should be informative, not decorative ("Account Settings" not "Your Space")
- Placeholder text: supplementary hint only, never a replacement for a label

## Error Messages
Good error message formula: **[What happened] + [Why] + [How to fix it]**
- ✅ "Password must be at least 8 characters and include a number."
- ✅ "We couldn't find an account with that email. Try a different email or create a new account."
- ❌ "Invalid input."
- ❌ "Error 422."
- ❌ "Something went wrong."

## Empty States
Good empty state copy formula: **[What this area is for] + [Why it's empty] + [What to do next]**
- ✅ "No projects yet. Create your first project to get started." + CTA button
- ❌ "Nothing here."

## Microcopy & Helper Text
- Hint text (below field): explain format, constraints, or why info is needed
  - "We'll only use this to send your receipt."
  - "Format: MM/DD/YYYY"
- Tooltips: for supplementary info only — primary info should be visible
- Confirmation messages: reassure users after a significant action
  - "Your settings have been saved." not just removing the save button

## Tone Consistency
- Match brand voice throughout (formal/casual/playful/authoritative)
- Don't shift tone mid-flow (casual onboarding → cold legal error messages)
