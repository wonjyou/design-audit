# Visual Design Reference

## Hierarchy
- Does a clear focal point exist? Is it the intended primary action or content?
- Are size, weight, and color used consistently to indicate importance?
- Can a user identify the H1 → H2 → body hierarchy in under 3 seconds?

## Spacing
- Is a consistent spacing scale used (e.g., 4px, 8px, 16px, 24px, 32px)?
- Are related elements grouped with less space than unrelated elements (Gestalt: proximity)?
- Is there adequate breathing room (white space) around key content and CTAs?
- Are padding values consistent within component types?

## Alignment
- Are elements aligned to a grid (8pt or 4pt grid recommended)?
- Is left-alignment consistent for body text blocks?
- Are interactive elements (buttons, inputs) aligned consistently across the layout?

## Typography
- Minimum readable body text: 16px web, 14px mobile (12px absolute minimum)
- Line height: 1.4–1.6× for body copy
- Max line length: 45–75 characters per line for readability
- Maximum 3–4 type sizes on a screen to avoid visual noise
- Sufficient contrast: WCAG AA = 4.5:1 normal text, 3:1 large text (18px+ or 14px+ bold)
- Font choice appropriate for context (avoid decorative fonts for body copy)

## Color
- Color not used as the ONLY means of conveying information (accessibility requirement)
- Brand colors used consistently and purposefully
- Sufficient contrast for all text and meaningful UI elements
- Semantic color use consistent: red = error/danger, green = success, yellow = warning

## Consistency
- Do similar elements look the same throughout? (buttons, cards, form fields)
- Is the design system / component library being followed (if applicable)?
- Are icon styles consistent (same line weight, filled vs. outline, size)?
